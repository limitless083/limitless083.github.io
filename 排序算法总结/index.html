简介

排序有内部排序和外部排序

  • 内部排序是数据记录在内存中进行排序。
  • 外部排序是因排序的数据很大，一次不能容纳全部的排序记录，在排序过程中需要访问外存。

常说的八大排序是内部排序。

分类

  • 选择排序
    • 简单选择排序
    • 堆排序
  • 插入排序
    • 直接插入排序
    • 希尔排序
  • 交换排序
    • 冒泡排序
    • 快速排序
  • 归并排序
  • 基数排序

直接插入 O(n^2) O(n^2) O(n) O(1) 稳定

稳定算法

  • 直接插入，冒泡，归并，基数

简单选择排序

在要排序的数组中选出最小的一个数，和第1个位置交换，在剩下的选最小的，和第2个交换，一直到最后一个。（选最大同）

示例：

  • 比较次数都是n(n-1)/2
  • 移动次数，最好有序 0 次，最坏第一个最大，后面有序3(n-1)，n-1次交换，每次交换移动3次
  • 时间复杂度都是O(n^2)，空间O(1)
  • 不稳定

直接插入排序

先将第一个元素作为有序，依次将后面的元素插入到有序序列中合适的位置，插入过程是不断向前移动的。

  • 时间复杂度O(n^2)，最好有序O(n)，最坏反序移动n(n-1)/2次，O(n^2)
  • 空间复杂度O(1)
  • 算法稳定，遇到相等的元素不移动

冒泡排序

依次将相邻元素比较，大的向后交换，一趟完成后最大数在最后，继续完成前n-i个相邻比较，以此类推。

优化：

  • 记录最后一次交换位置pos

  • 时间复杂度O(n^2)，最好有序O(n)，最坏反序O(n^2)

  • 空间复杂度O(1)
  • 算法稳定，相邻相等不交换

快速排序

每次将数组分成两部分，前一部分所有元素都小于后一部分，依次对各部分再次快排，直到结束。

思路：

  1. 选择第一个元素为key=A[0]
  2. 设置两个指针，i=0, j=N-1
  3. 从后向前搜索j--，找到A[j]<key，和A[i]交换
  4. 然后从前向后搜索i++，找到A[i]>key，和A[j]交换
  5. 重复3 4，直到i>=j结束
  6. 步骤3,4,5是一趟，然后对分成的两部分再次快排，直到结束

  • 时间复杂度平均O(nlogn)，最坏O(n^2)，每次划分出n-1和1个，相当于冒泡，最好O(nlogn)，每次划分n/2
  • 空间复杂度最好O(logn)，最坏O(n)，递归调用占用
  • 算法不稳定，利用辅助数组可实现稳定

归并排序

分治策略，不断拆分成子序列，一直到子序列有序，再不断合并成有序子序列，直到全部有序

合并过程需要辅助空间，先申请一个两个子序列之和大小的空间，然后每个子序列一个指针i,j，依次比较移动，放入新申请的空间

分而治之(divide - conquer)
每个递归过程涉及三个步骤：

  • 分解: 把待排序的 n 个元素的序列分解成两个子序列, 每个子序列包括 n/2 个元素
  • 治理: 对每个子序列分别调用归并排序MergeSort, 进行递归操作
  • 合并: 合并两个排好序的子序列,生成排序结果

  • 空间复杂度：O(n)，辅助空间，不是就地排序
  • 算法稳定

单链表-归并

  • 设置两个指针，一个步长为1， 一个步长为2，当快指针到达尾结点时，慢指针指向中间结点，时间复杂度为O(N)；
  • 平分为左链表L1和右链表L2，递归分裂，直到链表为空或者只有一个结点；
  • 将链表L2的每个结点插入到链表L1中，时间复杂度为O(m+n)，m、n分别为两条链表的长度。（合并两个有序链表）

堆排序

堆用一维数组来表示就是完全二叉树，且所有非叶节点的值都大于或等于其子女的值，根节点是最大值（最大堆）。

堆排思想

  1. 构建堆，将n个无序数组调整为堆，升序最大堆，降序最小堆
  2. 将堆顶元素（最大值）与数组倒数第i个元素交换，调整前n-i个元素为最大堆
  3. 依次重复步骤2，直到数组有序

图解排序算法(三)之堆排序

  • 父节点(i-1)/2
  • 左子节点2i+1
  • 右子节点2i+2

构建堆的过程
从倒数第一个非叶子节点开始，索引是length/2，如果比父节点大，就交换，一直向上比较。依次对每个非叶子节点执行Shift-Up

交换后重建过程
只需要调整堆顶元素，Shift-Down，如果比左右子节点小，则和较大的子节点交换，一直向下比较

  • 时间复杂度O(nlogn)，建堆过程O(n)，重建过程O(logn)，外层O(n)，最好最坏情况都是O(nlogn)
  • 空间复杂度O(1)，原地排序
  • 算法不稳定

希尔排序

希尔排序是插入排序的优化版，是按一定的增量一次分组进行直接插入排序，直到增量变为1，分成一组，全部调整为有序。优化点在于减少了直接插入的移动次数。

一般选择的增量序列是：n/2, n/4, n/8, ... , 1，希尔增量，不是最优的。

图解排序算法(二)之希尔排序

  • 时间复杂度，和增量选择有关，最优O(n^1.3)，最差O(n^2)
  • 空间复杂度，O(1)
  • 算法不稳定，各自分组插入会乱序

基数排序（计数排序）

不进行关键字的比较,而是利用”分配”和”收集”。

将根据整数的最右边数字将其扔进相应的0~9号的篮子里，对于相同的数字要保持其原来的相对顺序（确保排序算法的稳定性），然后将篮子里的数如图所示的串起来，然后再进行第二趟的收集（按照第二位的数字进行收集），就这样不断的反复，当没有更多的位时，串起来的数字就是排好序的数字。

基数排序与桶排序，计数排序【详解】

  • 时间复杂度，平均最好最差都是O(d*(n+r))，不需要比较
  • 空间复杂度，O(n+r)

其中，d 为位数，r 为基数，n 为原数组个数。

计数排序

计数排序的核心思想是，构建一个足够大的数组 hashArray[]，数组大小需要保证能够把所有元素都包含在这个数组上。元素的值为hashArray索引，hashArray[nums[i]]为数字出现次数。最后再按索引排，取出非零。

排序算法系列：基数排序

  • 空间需求很大，和数的范围有关
  • 时间复杂度为Ο(n+k)（其中k是整数的范围）

桶排序

桶排序的基本思想是将一个数据表分割成许多buckets，然后每个bucket各自排序，或用不同的排序算法，或者递归的使用bucket sort算法。也是典型的divide-and-conquer分而治之的策略。它是一个分布式的排序。

基本流程：

  • 建立一堆buckets；
  • 遍历原始数组，并将数据放入到各自的buckets当中；
  • 对非空的buckets进行排序；
  • 按照顺序遍历这些buckets并放回到原始数组中即可构成排序后的数组。
